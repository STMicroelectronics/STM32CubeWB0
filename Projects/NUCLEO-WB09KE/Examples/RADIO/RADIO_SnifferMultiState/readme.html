RADIO_SnifferMultiState Example Description

How to configure receiving packets on different channels using multiple state machines.

This code implements a sniffer application on different frequency channels, some of which are configured with encryption. Once the packet is received, the RSSI associated with the packet, the timestamp, and the entire frame are printed.

To implement this scenario, the ActionPacket mechanism is used. Each ActionPacket packet is configured with a different state machine. Each state machine has its own frequency channel, network address, wakeup time and encryption. In the DataRoutine, the code to schedule the next ActionPacket is placed.

How to use it?

In order to make the program work, you must do the following:

  • Open the project with your preferred toolchain
  • Rebuild all files and flash the board with the executable file
  • Run the example
