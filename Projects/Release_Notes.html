Release Notes for

Projects of STM32CubeWB0 Firmware Package

Copyright © 2024 STMicroelectronics

Purpose

The projects are a set of applicative tasks that aim to explain and provide use cases of the different products features based on the products hardware (Boards, STM32 Core(s) features, interconnections and peripherals) and built around the different firmware components. They can be directly used or adapted by the STM32CubeWB0 Firmware Package users. They provide basic functionalities, show main features or demonstrate performance. They are provided with preconfigured projects for the main supported toolchains.

In the STM32CubeWB0 Firmware Package, the projects are organized in a sequential scheme based on the following levels:

  • Boards : Hardware platform (nucleo boards)
  • Project Category:
    • Applications
    • Demonstrations
    • Examples
    • Examples_LL
    • Examples_MIX
    • Templates
    • Templates_LL

The exhaustive list of projects is provided in this table: STM32CubeProjectsList.

Known Limitations

  • HAL/LL Examples
    • Not applicable
  • MW STM32_BLE Applications
    • BLE_p2pServer application: if more than one client is connected, the notification message is sent only to the latest connected client.
    • FUOTA framework is not supported in STM32WB05xx devices due to memory constrains.
    • BLE_ApplicationInstallManager, STM32CubeIDE Debug configuration is not supported due to memory constrains.
    • Missing initialized event on BLE_TransparentMode and BLE_TransparentMode_C_O applications.
    • Missing preprocessor linker options on STM32CubeMX STM32_BLE projects with OTA service:
      • User is requested to manually add the following preprocessor linker options on IDE linker tab:
        • MEMORY_FLASH_APP_OFFSET=0x1000
        • MEMORY_FLASH_APP_SIZE=0x3E800 for STM32WB09xx
        • MEMORY_FLASH_APP_SIZE=0x1F000 for STM32WB07xx
    • Wrong build error generated on STM32CubeIDE, STM32_BLE projects due to the GNU Binutils v2.40.0.
      • User is requested to add the following preprocessor linker option on STM32CubeIDE linker tab: -z noexecstack
    • When generating an STM32_BLE application from scratch using the STM32CubeMX tool, user must add the stm32wb0x_nucleo_conf.h file on application folder Core. stm32wb0x_nucleo_conf_template.h reference file is available on Drivers32WB0x-nucleo folder.

Dependencies

  • STM32CubeMX V6.13.0
    • Projects (Applications and Examples) are generated using STM32CubeMX version V6.13.0.

Backward compatibility

  • STM32_BLE Applications are provided with the System folders files centralized on Projectfolder (recommended approach).
    • By default, STM32CubeMX v6.13.0 generates STM32_BLE projects with this new feature: user can generate STM32_BLE projects with System folder, by disabling the COMMON_BLE utility available on STM32CubeMX version V6.13.0.

Known Limitations

  • HAL/LL Examples
    • Not applicable
  • MW STM32_BLE Applications
    • BLE_p2pServer application: if more than one client is connected, the notification message is sent only to the latest connected client.
    • FUOTA framework is not supported in STM32WB05xx devices due to memory constrains.
    • STM32CubeIDE STM32_BLE applications are not provided due to memory constrains:
      • NUCLEO-WB05KZ: BLE_p2pServer_Ext, BLE_TransparentMode
    • Missing preprocessor linker options on STM32CubeMX STM32_BLE projects with OTA service:
      • User is requested to manually add the following preprocessor linker options on IDE linker tab:
        • MEMORY_FLASH_APP_OFFSET=0x1000
        • MEMORY_FLASH_APP_SIZE=0x3E800 for STM32WB09xx
        • MEMORY_FLASH_APP_SIZE=0x1F000 for STM32WB07xx
    • Wrong build error generated on STM32CubeIDE, STM32_BLE projects due to the GNU Binutils v2.40.0.
      • User is requested to add the following preprocessor linker option on STM32CubeIDE linker tab: -z noexecstack
